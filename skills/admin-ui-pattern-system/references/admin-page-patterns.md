# 后台页面范式参考

本文件用于在具体页面设计时加载。先根据业务任务选择主范式，再结合当前模板能力落地。

## 标准列表管理页

适用：用户、角色、任务、配置对象、策略、批次、运行记录等可搜索、分页、批量处理的对象。

默认结构：

- 顶部或卡片内搜索区：只放高频条件，低频条件折叠。
- 主表格：默认展示 P0 字段。
- 表格工具栏：新增、批量操作、刷新、全屏、列设置、密度等能力按模板已有能力复用。
- 行操作：最高频操作直接展示，其他进入更多菜单。
- 详情：默认使用抽屉；信息过多、需要多 Tab 或深链时使用详情页。
- 创建/编辑：默认使用抽屉；短确认或极轻量输入才使用弹窗；复杂流程使用步骤页或独立页。

适合保留在主表字段：

- 名称/标题
- 状态
- 类型/分类
- 归属/范围
- 关键时间
- 风险等级或处理状态
- 少量高频操作

应进入详情或可选列：

- 长说明
- 原始 ID、trace、permission decision
- raw payload
- 审计历史
- 低频业务字段

## 树表主从页

适用：组织、部门、分类、本体分类、菜单、权限范围、数据域等层级结构。

默认结构：

- 左侧树/分类面板：搜索、展开/收起、节点操作。
- 右侧表格/详情：展示当前节点下对象。
- 节点操作和对象操作分离。
- 节点详情、对象详情、编辑表单默认通过抽屉承载；简单确认才用弹窗。

避免：

- 左侧树、右侧多个列表、下方表单同时堆叠。
- 节点配置和对象配置混在同一个表单里。

## 配置治理页

适用：规则、策略、字段映射、状态裁定、动作目录、权限策略、schema、本体版本。

默认结构：

- 配置对象列表：状态、版本、发布状态、风险等级。
- 详情/编辑入口：默认进入抽屉；字段多、影响范围复杂或需要 diff/审批上下文时进入独立详情页。
- 草稿编辑：独立编辑流，避免在列表里直接改高风险字段。
- diff/预览：发布前展示影响范围。
- 发布/回滚/禁用：高风险动作必须确认并审计。

必须表达：

- 当前线上版本
- 草稿状态
- 最近发布时间
- 修改人/审批人
- 影响范围
- 回滚路径

## 审批/流程处理页

适用：审批、复核、dry-run、批次执行、流程实例、异常处理。

默认结构：

- 队列/实例列表：状态、风险、创建时间、申请人/范围。
- 筛选区：状态、风险、时间、对象类型。
- 详情区域：证据、上下文、影响范围、历史操作。
- 操作区：批准、拒绝、重试、取消、提交执行等。
- 审计轨迹：抽屉、时间线或详情页内展示。

避免：

- 把所有动作计划都变成表格行内按钮。
- 执行动作缺少影响范围和后果说明。
- 只展示成功路径，不展示失败/部分失败。

## 图谱/画布工作台

适用：图谱编辑、本体编辑、流程编排、拓扑关系、节点边编辑。

默认结构：

- 中心画布：占据主要空间。
- 顶部工具栏：保存草稿、撤销/重做、适配视图、发布、回滚、diff。
- 左侧可选：对象列表、节点库、图层/分类。
- 右侧 inspector：选中节点/边后的属性、关系、风险、审计信息。
- 新增节点/关系：优先用右侧 inspector 或抽屉承载；短选择器可用弹窗；复杂关系创建用专门流程。
- 图例/帮助：折叠、弹出或次级面板，不常驻挤压画布。

必须避免：

- 画布很小，右侧表单和说明长期占据主要空间。
- 添加节点、添加关系、图例、说明、属性编辑全部同时显示。
- 手写简陋 SVG 作为生产级编辑器。

## 审计/观测页

适用：审计日志、权限决策、执行记录、任务轨迹、系统事件、错误记录。

默认结构：

- 时间、状态、用户、范围、对象类型筛选。
- 事件表格：紧凑、可扫描。
- 详情抽屉：完整 payload、上下文、关联对象；超长轨迹或多对象关联可升级为独立页。
- 轨迹视图：需要时用时间线或链路详情。
- 导出/复制：按权限开放。

注意：

- 原始 ID 和 payload 不应淹没默认列表。
- 失败原因、决策结果、风险等级要比底层字段更容易看到。
