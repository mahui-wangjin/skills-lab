Architecture Review Surface

Main Boundary

主流程仍在 SKILL.md 和 delivery-gates.md；报告契约拆到 references/human-validation-report.md；稳定脚手架放到 scripts/create_report.py。

Dependency Direction

交付主流程引用报告 reference；报告 reference 可调用脚本；脚本只生成骨架，不反向决定交付是否完成。

Non-goals

本次没有给所有任务强制生成完整报告，没有新增 CI workflow，也没有让截图替代自动化测试。

Core Logic Review Surface

  1. 任务进入 PDM 后先按简单/中等/复杂/生产级分流。
  2. 中等及复杂任务最终输出 Human Validation Packet。
  3. 符合报告触发条件时，使用脚本生成 HTML 报告骨架，再填入真实证据。
  4. 证据地图把本地验证、CI/workflow、浏览器截图、日志和审查结论分级，并写明限制。
  5. steelman review 必须挑战报告是否真的让人能验收。

Invariant: 不得把未运行的 CI 写成已通过；不得把截图过度解释为后端正确性；不得因为生成了报告就宣称生产无风险。

Steelman Counter-Review

Strongest objection 1: The report requirement could make every small task heavy. Decision: addressed by keeping full HTML reports conditional and allowing small tasks to use a compressed handoff.

Strongest objection 2: A scaffold script could create false confidence if placeholders remain. Decision: addressed by documenting that placeholders must be replaced and by keeping final completion tied to evidence, not report existence.

Strongest objection 3: CI/workflow evidence might be confused with user acceptance. Decision: addressed by explicitly defining CI as evidence input and requiring "proves / does not prove" mapping.

Strongest objection 4: The report template may become hard to maintain if embedded in SKILL.md. Decision: addressed by moving the detailed contract to a reference file and deterministic scaffold script.

Remaining risk: Trigger quality still depends on future agents reading the reference before final handoff. This is mitigated by adding the reference callout to SKILL.md and delivery-gates completion standards.
